## Context

分帳工具使用 ImmersiveToolScaffold 架構，目前只支援等分模式（總金額 ÷ 人數）。有分享卡片功能。計算邏輯是純函式 `calculateSplitBill()`。

## Goals / Non-Goals

**Goals:**

- 新增小費百分比選項（0/5/10/15/20%），加到總額後再分帳
- 新增不等分模式（自訂比例），每人可設不同比例
- 新增多項目拆分模式，每項目獨立金額和參與人數

**Non-Goals:**

- 不支援幣別選擇（台幣為主）
- 不支援歷史記錄
- 不改動分享卡片樣式

## Decisions

### 小費百分比整合到等分模式

在現有等分模式的總金額下方新增 Chip 行：0% / 5% / 10% / 15% / 20%。選擇後自動計算小費金額並顯示。分帳計算的 total 變成 `原始金額 + 小費`。分帳結果顯示小費金額和最終總額。

### 不等分模式 — 比例輸入

新增模式切換（SegmentedButton：等分 / 比例 / 多項目）。比例模式下，每人顯示一個比例輸入欄（預設 1），系統根據各人比例計算佔比。例如 3 人比例 1:2:1，總額 4000 → 1000/2000/1000。

### 多項目拆分模式

使用者新增多個項目（品名 + 金額 + 參與人數），每個項目獨立計算等分，最後彙總每人總計。適合聚餐場景：某些菜大家分、某些菜特定人分。

### 計算邏輯擴展

保持現有 `calculateSplitBill()` 不變（向後相容）。新增：
- `calculateWithTip()` — 加入 tip percentage
- `calculateByRatio()` — 按比例分帳
- `calculateMultiItem()` — 多項目彙總

## Risks / Trade-offs

**[UI 複雜度增加]** → 用 SegmentedButton 三模式切換，每個模式獨立 UI section，不會互相干擾。

**[多項目模式 UX 較重]** → 預設只顯示等分模式，進階模式需要主動切換。
