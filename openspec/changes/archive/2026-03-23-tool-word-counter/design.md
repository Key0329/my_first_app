## Context

工具箱已有 18 個工具，透過 tool-registry（`allTools` in `tool_item.dart`）自動註冊路由。新增工具只需：(1) 建立工具頁面、(2) 在 `allTools` 新增 `ToolItem`、(3) 新增 i18n 字串。所有工具頁面使用 `ImmersiveToolScaffold` 作為共用骨架。

文字計數器是純前端工具，不需要網路 API 或外部套件，只需即時分析使用者輸入的文字。

## Goals / Non-Goals

**Goals:**

- 提供即時的文字統計（字元數、字數、行數、段落數）
- 支援中英文混合計數（中文以字為單位、英文以空白分詞）
- 顯示預估閱讀時間
- 遵循現有工具開發模式，新增一個 `ToolItem` 即完成整合
- 完整 i18n 支援（繁體中文 + 英文）
- 撰寫 widget test

**Non-Goals:**

- 不做字頻分析或關鍵字提取
- 不做文件匯入（.txt, .docx 等）
- 不做拼字檢查或文法建議
- 不做雲端同步或歷史紀錄

## Decisions

### 文字計數邏輯

使用純 Dart 實作，不引入外部 NLP 套件：
- **字元數**：`String.length`（含空白）+ 不含空白版本
- **字數**：以 RegExp 拆分，中文字元每個算一個字，英文以空白分詞
- **行數**：以 `\n` 分割計算
- **段落數**：以連續空行（`\n\n`+）分割，過濾空段落
- **閱讀時間**：中文字 300 字/分鐘 + 英文字 200 字/分鐘，取上限至少 1 分鐘（< 1 分鐘顯示「< 1 分鐘」）

**理由**：文字計數器功能明確且演算法簡單，無需引入重量級套件。

### UI 佈局

- 上方：全寬 `TextField`（multiline, maxLines: null），佔畫面主要區域
- 下方：統計結果以 Grid 卡片形式展示（2×3 網格）
  - 字元數（含空白）
  - 字元數（不含空白）
  - 字數
  - 行數
  - 段落數
  - 閱讀時間
- AppBar 右上角：清除按鈕（清空文字）+ 複製按鈕（複製統計摘要）

**理由**：Grid 卡片佈局與現有工具（如 BMI 結果區）風格一致，一目了然。

### 即時計算策略

在 `onChanged` 回調中直接計算統計，不使用 debounce。

**理由**：純字串操作在數千字內不會有效能問題，即時回饋體驗更佳。

## Risks / Trade-offs

- [超長文字效能] 若使用者貼入數萬字文本，`onChanged` 每次觸發完整計算可能造成卡頓 → 短期可接受，未來可加 debounce 或 Isolate 處理
- [中英混合計數精度] 簡單的 RegExp 分詞無法完美處理所有邊界情況（如全形標點、emoji） → 對工具箱定位而言精度足夠
